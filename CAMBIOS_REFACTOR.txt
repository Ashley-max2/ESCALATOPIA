REGISTRO DE CAMBIOS Y REWORK (SOLID) - ESCALATOPIA
==================================================

RESUMEN DE CAMBIOS TÉCNICOS
---------------------------
Se ha realizado una reestructuración completa del código para eliminar la "deuda técnica" y cumplir con principios SOLID, manteniendo la funcionalidad original descrita en el GDD.

1. ARQUITECTURA DEL JUGADOR (NUEVA)
   - Eliminado: Se borró `PlayerControllerRefactored.cs` y scripts monolíticos antiguos.
   - Implementado: Patrón State Machine (Máquina de Estados).
     * `PlayerStateMachine.cs`: Contexto principal.
     * `PlayerState.cs` & `PlayerStateFactory.cs`: Gestión limpia de estados.
     * Estados: Grounded, Jump, Air, Climb, Hook.
     * `PlayerInputHandler.cs`: Separación de lógica de entrada.
   - Beneficio: Facilita añadir nuevas mecánicas sin romper las existentes (Open/Closed Principle).

2. SISTEMA DE UI (NUEVO)
   - Creado sistema modular basado en Vistas (`UIView`).
   - `UIManager`: Singleton para manejar pantallas.
   - `DialogueView`: Ventana específica para diálogos del tutorial.
   - Editor Tool (`UITools.cs`): Automatiza la creación del Canvas y registro de vistas.

3. CORRECCIONES EN BOSSES & GOAP
   - Se arreglaron errores de compilación en `GOAPService.cs`.
   - Se añadieron métodos faltantes en `GOAPAction.cs` (`CheckPreconditions`, `Effects`) para que la IA de los jefes funcione correctamente.

4. SISTEMA DE GANCHO (HOOK)
   - Se limpió la duplicidad de scripts. Se eliminaron versiones "Refactored" incompletas (`HookSystemRefactored`) y se mantuvo el `HookSystem.cs` original validado, integrándolo correctamente con la State Machine del player.

5. HERRAMIENTAS DE EDITOR (NUEVAS)
   - `TutorialBuilderWindow.cs`: Herramienta para generar el nivel procedimentalmente.
   - Solución de "Material Leaks": Se implementó un sistema seguro de creación de materiales en modo Editor para evitar errores de memoria.

--------------------------------------------------

PASOS PENDIENTES / QUÉ HACER AHORA
----------------------------------

1. ASIGNACIÓN DE TAGS
   - Unity puede haber perdido tags si no estaban definidos.
   - Ve a "Edit > Project Settings > Tags and Layers".
   - Asegúrate de tener creados los tags: "escalable", "HookPoint", "respawn", "Player".

2. PROBAR INPUTS
   - El sistema usa `Input.GetAxis("Horizontal")`, "Vertical" y "Jump".
   - Asegúrate de que en "Project Settings > Input Manager" estos ejes están definidos (suelen estarlo por defecto en Unity).

3. INTEGRACIÓN VISUAL
   - Los objetos generados son primitivos (cubos, esferas).
   - Para la versión final, reemplaza los MeshFilter de "Pared_Escalada" o "Platform" por tus modelos 3D definitivos.

4. VALIDAR BOSSES
   - Como se tocó la base del GOAP, abre la escena de un Boss y verifica que compila y se mueve. Los scripts lógicos están arreglados, pero verifica las referencias en el Inspector.

5. LIMPIEZA
   - Puedes borrar la carpeta "Assets/Scripts/Player/StateMachine/StatesRefactored" si todavía existe vacía, ya que todo se movió a la estructura limpia.
